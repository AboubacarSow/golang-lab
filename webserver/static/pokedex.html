Pokedex — REPL PokeAPI Explorer

A REPL CLI that talks to the PokeAPI, caches data, and manages a local pokedex with inspect and catch commands.

Overview

The Pokedex project is an interactive command-line explorer for Pokémon location areas and captures. It demonstrates HTTP client usage, cache TTL handling, mutex synchronization, and REPL design in Go.

Features

  • Explore PokeAPI location areas
  • List and catch Pokémon with probability-based capture
  • Inspect caught Pokémon details
  • Local pokedex storage and paginated navigation

Concepts

  • HTTP client integration with net/http
  • JSON decoding and API modeling
  • Concurrent cache cleanup with goroutines
  • Mutex-protected cache state
  • REPL command handling and user interaction

Run locally

cd ../pokedex
go run .

Structure

  • main.go — program entry and REPL setup
  • repl.go — interactive engine and command loop
  • internal/pokeapi/ — PokeAPI client and request handling
  • internal/pokecache/ — cache layer with TTL and mutex

Why it matters

This project is ideal for learning real-world API integration in Go while keeping the user experience interactive. It also shows how caching and concurrency can improve the responsiveness of a CLI tool.
