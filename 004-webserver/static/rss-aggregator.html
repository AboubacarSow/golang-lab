rss-aggregator — RSS Aggregator API Service

A backend service that stores RSS feeds, users, and feed follows in PostgreSQL while scraping feed posts on a background schedule.

Overview

This project demonstrates a REST API built with chi, typed database access via sqlc, and a background scraper that parses RSS feeds into a Postgres database.

Key components

  • User management with generated API keys
  • Feed creation and listing
  • Follow feeds to collect posts
  • Background scraping workers for RSS polling
  • PostgreSQL persistence for feeds, follows, and posts

Architecture

The service routes requests through auth middleware, maps handlers to sqlc queries, and runs concurrent scraper workers to fetch RSS feeds in the background.

API Endpoints

POST /api/v1/users
GET /api/v1/users
DELETE /api/v1/users/{id}
POST /api/v1/feeds
GET /api/v1/feeds
DELETE /api/v1/feeds/{id}
POST /api/v1/feed_follows
GET /api/v1/feed_follows
GET /api/v1/posts

Run locally

cd ../rss-aggregator
# create a .env file with PORT and CONNECTION_STRING
go run main.go

Structure

  • main.go — server entry point and route setup
  • scraper.go — RSS fetch scheduler and worker coordination
  • rss.go — RSS XML parsing
  • internal/database — sqlc-generated DB layer
  • migrations/ — Postgres schema and query definitions

Why it matters

rss-aggregator is a practical example of combining API design, persistent storage, and background processing in Go.
