Inventory — Terminal Inventory Manager

A TUI-based inventory app built with clean architecture, JSON persistence, and a layered internal package structure.

Overview

This project demonstrates how to build a terminal UI inventory manager in Go using a separated domain layer, storage layer, and UI layer. The app stores data in data/inventory.json and renders a table-based interface in the terminal.

Features

  • View inventory in a table layout
  • Add new stock items with quantities
  • Delete items by index
  • Persistent storage using JSON

Concepts

  • Clean architecture and package separation
  • Terminal UI with tview
  • File-based persistence and JSON encoding
  • Domain logic inside internal/inventory

Run locally

cd ../inventory
go run ./cmd

Structure

  • cmd/ — application entry point
  • data/ — JSON storage file
  • internal/inventory/ — domain logic and storage
  • internal/ui/ — terminal UI layer

Why it matters

This page shows how separating business logic from presentation makes a terminal app easier to maintain and extend. The Inventory project is a great example of applying SRP and package design in a small Go system.
